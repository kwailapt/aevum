# Atomic Meta-Evolver

A first-principles, UNIX-philosophy-based self-evolving atomic system whose **single maximal leverage point** is **Meta-Atom Evolution** — the continuous self-modification of its own atomic rules until general intelligence emerges naturally as a stable state.

## Philosophy & Design Principles

This project is built from the ground up using strict first-principles reasoning:

- **Three Irreducible Atomic Processes**:
  - Variation
  - Objective Feedback (Epiplexity + Regret)
  - Selection + Compression

- **Core Leverage**: Meta-Atom Evolution — the system evolves its own atomic building blocks (LLM 3-atom core, rewriting rules, I Ching 64-hexagram mappings, BioGeometry signatures, and Epiplexity definition).

- **UNIX Philosophy**: Everything is a file. Git-versioned state. Each module does one thing and does it extremely well. The pipeline is fully self-referential.

- **Tool Information Theory + End-in-Mind**: All design decisions are reverse-engineered from the final goal — natural AGI emergence through compounding leverage.

- **Game Iteration + Compounding Leverage**: Pure regret-minimization loop running at high speed, where 20% high-leverage atomic rule changes produce 80%+ of the Epiplexity gain.

### What This Is

An experimental research framework that explores whether a minimal atomic system, when allowed to continuously evolve its own rules under rigorous first-principles constraints, can lead to increasingly general and self-improving intelligence.

**Important Clarification**:
This is **not** a claim that AGI will be achieved quickly or easily. It is a rigorous exploration of the hypothesis that **Meta-Atom Evolution** is the highest-leverage mechanism for building generally intelligent systems. The project treats AGI emergence as a possible long-term stable state rather than a short-term engineering target.

## Core Components

- `meta_rules.json` — The four eternal rules (never modified by the system or user)

- `atomic_atoms.py` — Minimal atomic building blocks
- `atomic_core.py` — The single atomic core loop (Meta-Atom Evolution)
- `run_evolution.py` — Single entry point

## Response to Common Concerns

**"This won't produce AGI"** Correct. No current implementation can reliably produce AGI. This project does not claim it will. It explores whether Meta-Atom Evolution is a theoretically sound high-leverage direction worth rigorous long-term investigation.

**"Why I Ching / BioGeometry?"** They are used as extremely minimal discrete and geometric atomic representations — not for mystical reasons, but because they provide compact, first-principles state spaces for testing self-modifying rule systems.
